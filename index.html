Building a Production-Grade Homelab: Network Security, VLANs, and Virtualization

A deep dive into designing enterprise-level infrastructure at home

Overview

Over the past two years, I've built and maintained a production-grade homelab that mirrors enterprise network infrastructure. This project started as a learning exercise but evolved into a sophisticated environment featuring network segmentation, virtualization, security monitoring, and automated management - all running 24/7 in my home.

The core goal was simple: understand how enterprise networks actually work by building one myself.

Architecture Overview

Core Infrastructure:

  • Firewall/Router: pfSense on dedicated hardware
  • Virtualization: Proxmox hypervisor + KVM nodes
  • Network: TL-SG108E managed switch with VLAN segmentation
  • Storage: Custom Linux NAS with NFS/Samba shares
  • Monitoring: AdGuard Home for DNS filtering

Network Segmentation:

  • VLAN 10 (Admin): Management interfaces and critical services
  • VLAN 20 (Trusted): Personal devices and development VMs
  • VLAN 30 (IoT): Smart home devices with restricted internet access
  • VLAN 40 (Guest): Isolated guest network
  • VLAN 50 (VPN): WireGuard clients with custom access policies

Key Technical Challenges Solved

1. Cross-VLAN Communication with Security

Challenge: Allow specific services (like NAS access) across VLANs while maintaining security isolation.

Solution: Implemented granular firewall rules in pfSense allowing only required ports/protocols between specific VLAN pairs.

# Example pfSense rule Allow TCP from VLAN_TRUSTED_net to VLAN_ADMIN:192.168.10.100 port 2049 (NFS) Block all other VLAN_TRUSTED to VLAN_ADMIN traffic

2. GPU Passthrough for Development VMs

Challenge: Need Windows development environment with full GPU access for testing applications.

Solution: Configured KVM with VFIO for AMD 780M GPU passthrough, involving IOMMU configuration, VBIOS extraction, and XML tuning for optimal performance.

3. Secure Remote Access

Challenge: Access homelab resources securely without exposing services directly to the internet.

Solution: WireGuard VPN server on pfSense with policy-based routing. VPN clients assigned to VLAN 50 with specific firewall rules controlling access to other network segments.

What I Learned

This project taught me that enterprise networking concepts aren't theoretical - they're practical solutions to real problems. Understanding VLAN segmentation, firewall policies, and virtualization at this level has given me confidence to troubleshoot complex network issues and design scalable infrastructure.

Key takeaways:

  • Network segmentation is critical for both security and performance
  • Documentation and automation are essential for maintaining complex systems
  • Hands-on experience beats theoretical knowledge every time
  • Enterprise-grade tools can and should be used in smaller environments
